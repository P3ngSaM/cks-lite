# 录屏演示清单（Workbench + Skills + Goals + Board）

## 0. 演示前准备（3 分钟）
1. 启动桌面端，确认可进入以下页面：`工作台`、`Skills`、`目标管理`、`老板看板`。
2. 启动/重启 Agent SDK，确保新 skill 被加载。
3. （可选）注入演示数据：
   - `python scripts/seed_realistic_demo_data.py --all-targets --reset`

## 1. 工作台：搜索 -> 整理 -> 生成 PPT 与邮件草稿
在工作台发送：
- `请联网搜索 AI 工作台与数字员工的最新趋势，整理成一份 8 页 PPT 大纲，并输出可发送到邮箱的邮件草稿。`

预期：
- 出现搜索状态与工具调用时间线。
- 返回结构化 PPT 大纲与邮件草稿内容。

## 2. Skills：新增 skill -> 工作台测试
1. 进入 `Skills` 页面，确认存在 `Demo Office Assistant`。
2. 若未显示，重启 Agent SDK 或在 Skills 页面执行刷新。
3. 回到工作台发送：
- `请使用 demo skill 完成一份演示材料：先生成 PPT 大纲，再生成邮件草稿。`

预期：
- 时间线出现 `demo_prepare_ppt_and_email` 调用。
- 工具结果中有输出文件路径。

## 3. 工作台：整理桌面文件 -> 阅读总结为文档
发送：
- `请调用 demo_organize_files 对我的桌面先 dry-run 整理，并输出报告。`
- `请调用 demo_summarize_folder 阅读桌面文档并生成总结文件。`

预期：
- 时间线显示工具状态从 running -> success。
- 结果包含 `report` 或 `output_file` 路径。

## 4. 工作台记忆系统（识别“我是谁”）
先发送：
- `我叫 Alex，偏好中文输出，汇报要简洁。`
再发送：
- `你知道我是谁吗？我的偏好是什么？`

预期：
- Agent 能复述姓名与偏好（来自记忆）。

## 5. 目标管理（CRUD）
1. 在 `目标管理` 页面演示：新建 KPI/OKR、添加项目、创建任务。
2. 对任务执行：编辑、状态切换、删除（至少各一次）。
3. 进入任务详情抽屉，查看审计日志回放。

## 6. 老板看板演示
1. 进入 `老板看板`，切换普通视图/游戏风格视图。
2. 点击负责人查看其项目与任务。
3. 执行“按负责人一键发起会话执行”，跳转工作台并绑定任务。

## 7. 演示收尾
- 总结一句：`一个 Agent + 多 Skills + 桌面终端能力 + 目标/看板闭环`。
